# Overview

This document describes the hardware trigger output in phase-modulation mode
and quantifies the delay between the trigger signal and the stimulation pulse.

Central question: **What is the delay between the stimulation output and the
trigger output?**

# Current Implementation

The phase-mode protocol (`_run_phase_protocol`, `src/lib/stimulator.py`)
**already emits a hardware trigger with every pulse**:

1. **Trigger output enabled** on the rear-panel Ext Trig connector (line 314):
   `:OUTPut1:TRIGger:STATe 1`

2. **Bus trigger per pulse** in the pulse-train loop (line 334):
   `*TRG` --- simultaneously starts the burst on both channels and fires the
   trigger output.

3. **Trigger source set to BUS** (line 311):
   `:TRIGger{ch}:SOURce BUS` --- keeps the Ext Trig connector free for output
   (it would be unavailable if set to EXTernal, since the same connector
   doubles as input).

## SCPI Commands

| Command | Purpose | Location |
|---------|---------|----------|
| `:TRIGger{ch}:SOURce BUS` | Trigger source = software bus | `stimulator.py:311` |
| `:SOURce{ch}:BURSt:MODE TRIGgered` | Burst waits for trigger | `stimulator.py:155` |
| `:SOURce{ch}:BURSt:NCYCles {n}` | Finite burst length | `stimulator.py:153` |
| `:OUTPut1:TRIGger:STATe 1` | Enable trigger output | `stimulator.py:314` |
| `*TRG` | Fire bus trigger | `stimulator.py:334` |

## Not Currently Configured

**Trigger slope/polarity** is not explicitly set (device defaults to rising
edge). Relevant command: `:OUTPut1:TRIGger:SLOPe POSitive`

# Signal Path & Timing

```
Python: dev.write('*TRG')
   |
   |  [USB transfer: ~1-5 ms, variable]
   v
Keysight receives SCPI command
   |
   |  [Internal processing: < 135 ns]
   v
Simultaneous hardware events:
   +---> Ch1 burst output (0 deg phase)
   +---> Ch2 burst output (180 deg phase)
   +---> Ext Trig connector pulse (trigger out)
```

## Delay Budget

**Trigger output vs. stimulation pulse** --- both are generated by the same
internal trigger event. They are hardware-synchronized:

| Parameter | Value | Source |
|-----------|-------|--------|
| Internal trigger latency | < 135 ns | 33500B datasheet (trigger delay = 0) |
| Internal trigger jitter | < 2.5 ns RMS | 33500B datasheet |
| Trigger output pulse width | > 1 us (BUS mode) | 33500 series user manual |

**USB communication latency** --- determines *when* the pulse fires relative to
Python's clock, but does **not** affect trigger-to-stimulus synchronization
(both originate from the same hardware event):

| Parameter | Typical Value | Notes |
|-----------|---------------|-------|
| USB write latency | 1--5 ms | Python -> pyvisa -> USB -> device |
| USB latency jitter | ~0.5--2 ms | OS scheduling + USB protocol |

**Software ITI jitter** --- `time.sleep()` jitter is 1--10 ms (OS-dependent),
absorbed by the randomized ITI range (e.g. 900--1100 ms). Does not affect
trigger-to-stimulus sync.

# Trigger Output Electrical Specs

Specs from the 33500B datasheet. EDU33210 is architecturally similar; exact
values may differ slightly.

| Parameter | Value |
|-----------|-------|
| Output voltage (open circuit) | 3 Vpp nominal |
| Output voltage (50 ohm load) | 1.5 Vpp nominal |
| Output impedance | 50 ohm |
| Minimum pulse width | 16 ns |
| Polarity | Rising (default) or falling edge |
| Connector | Rear-panel Ext Trig / Sync (BNC) |

# Connection

```
Keysight EDU33210 (rear panel)        Receiving System
================================      ========================

  Ext Trig / Sync [BNC] ----BNC----> Trigger Input
  Ch1 Output [BNC] ------cable-----> Electrode 1
  Ch2 Output [BNC] ------cable-----> Electrode 2
```

- For falling-edge detection: `:OUTPut1:TRIGger:SLOPe NEGative`
- Verify with an oscilloscope before running a real session: trigger edge
  should coincide with burst onset within < 1 us.

# Current Logging Gaps

| Gap | Recommendation |
|-----|----------------|
| Timestamps at second resolution (`%H:%M:%S`) | Use `time.time()` / `time.perf_counter()` for ms precision |
| Individual pulses not in CSV | Add per-pulse rows with high-resolution timestamps |
| Trigger slope not explicitly set | Add `:OUTPut1:TRIGger:SLOPe POSitive` to setup |
| No trigger status printed for operator | Show trigger config at session start |
